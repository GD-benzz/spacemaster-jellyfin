# 空间大师 · 安装与配置指南（Jellyfin 版）

> 这份文档是给**终端用户**看的。照着一步一步做，你就能在一台 Linux 主机（NAS / 服务器 / 迷你 PC）上，
> 用 Docker 跑起带「房间声学校正」的 Jellyfin，并在网页控制台输入房间尺寸、拖动滑块实时调音。
>
> 你看不到任何算法源码，也不需要任何人手动发给你文件——校正引擎随一个**公开的 tiny 镜像**分发，你只需一条命令把它取回本机。

---

## 〇、这套系统是什么（先看一眼）

空间大师 **不是** 一个独立播放器，它是 **Jellyfin 媒体服务器的一个改装层**：

```
你的房间尺寸（网页控制台 :8777）
        │
        ▼
  音频引擎 sm_dsp_engine  ──►  把 EQ / 延时 / 平衡 写到 /opt/spacemaster
        │
        ▼
  Jellyfin 转码  ──►  ffmpeg 薄壳读配置，向引擎要滤镜串并注入
        │
        ▼
   校正后的声音送到你的播放器
```

- **Jellyfin**（媒体服务器、播放、媒体库管理）—— 由你从 Docker Hub 自己拉取，**不捆绑在我们的镜像里**。
- **空间大师引擎**（房间校正算法）—— 编译后的二进制，随公开镜像 `ghcr.io/gd-benzz/spacemaster-engine` 分发。
- **两个端口**：`8096` = Jellyfin 网页；`8777` = 校正控制台网页。

---

## 一、你需要准备什么

| 项目 | 说明 |
|------|------|
| 一台 Linux 主机 | NAS（飞牛/群晖/威联通等）、服务器、迷你 PC 都行，能装 Docker。 |
| 局域网 IP | 这台主机的 IP，下文记作 `<宿主IP>`（例如 `192.168.31.155`）。 |
| 登录方式 | 能 SSH 上去，或直接在它本机终端操作。 |
| 网络 | 能访问 Docker Hub 与 `ghcr.io`（都是公开镜像源）。 |

> 不需要：编译环境、算法源码、任何人私下传文件。

## 二、连接你的主机

用你的电脑连上那台主机（后面所有命令都在**那台主机**上执行）：

```bash
ssh 你的用户名@<宿主IP>
# 例如： ssh ben@192.168.31.155
```

连上后，后面的命令都在这台主机里敲。

## 三、安装 Docker 与 Docker Compose

### 方案 A：Ubuntu / Debian / 飞牛（类 Debian）

```bash
curl -fsSL https://get.docker.com | sudo sh
sudo systemctl enable --now docker
```

装好后验证：

```bash
docker --version
docker compose version      # 注意是 "docker compose"，中间有空格（Compose v2）
```

两行都应正常输出版本号，才算装好。

### 方案 B：群晖 / 威联通等 NAS 系统

- **群晖**：在「套件中心」装「Container Manager」（即 Docker），然后在里面用「项目」导入 `docker-compose.yml`。
- **威联通**：在「App Center」装「Container Station」。
- 这类图形界面 NAS 也可走 SSH 用方案 A 的命令装原生 Docker，二选一即可。

## 四、取回空间大师引擎（核心一步）

空间大师的校正算法是**编译后的二进制**，随公开镜像 `ghcr.io/gd-benzz/spacemaster-engine` 分发。
你只需要一条命令，把它从镜像里「抠」到你主机的 `/opt/spacemaster` 目录——**不用编译、不用任何人发文件给你**。

```bash
sudo mkdir -p /opt/spacemaster
sudo docker run --rm -v /opt/spacemaster:/out \
  ghcr.io/gd-benzz/spacemaster-engine:latest \
  sh -c 'cp /spacemaster/* /out/ && chmod +x /out/sm_dsp_engine /out/setup.sh'
```

执行后确认文件到位：

```bash
ls -la /opt/spacemaster
```

你应该看到至少这三个文件：

| 文件 | 作用 |
|------|------|
| `sm_dsp_engine` | 校正算法引擎（闭源二进制，14MB 左右） |
| `sm_wrapper.sh` | 顶替 Jellyfin 自带 ffmpeg 的薄壳脚本 |
| `setup.sh` | 容器启动时把 ffmpeg 换成薄壳的引导脚本 |

> 这一步是**唯一**会接触到闭源二进制的地方。取回后它一直躺在 `/opt/spacemaster`，Jellyfin 容器挂载它即可，之后无需再管。

## 五、拉取官方 Jellyfin（你自己拉，不捆绑）

底层媒体服务器由你自己从 Docker Hub 拉取，我们的镜像**不打包 Jellyfin**：

```bash
docker pull jellyfin/jellyfin:latest
```

## 六、取回集成层脚本（开源部分）

空间大师的开源脚本（启动配置、控制台界面）在 GitHub 公开仓库。下载到本机任意目录：

```bash
git clone https://github.com/GD-benzz/spacemaster-jellyfin.git
cd spacemaster-jellyfin
```

> 若你不想用 git，也可以只手动下载这三个文件放到同一目录：
> `docker-compose.yml`、`peq_toggle_nas.py`、`engine_runtime.py`（仓库根目录都有）。

## 七、启动 Jellyfin + 校正层

在刚才 `cd` 进来的仓库目录里执行：

```bash
docker compose up -d
```

这条命令会：

1. 用你在第五步拉的官方 `jellyfin/jellyfin` 镜像建容器（容器名 `jellyfin-sm`）；
2. 通过 `setup.sh` 在容器启动时，把 Jellyfin 自带的 `ffmpeg` 换成我们的校正薄壳；
3. 把宿主的 `/opt/spacemaster` 挂进容器，让薄壳能找到引擎二进制；
4. 把 `./config`、`./cache` 挂进容器，做 Jellyfin 的配置与转码缓存。

> **若你之前装过旧版本**（容器名也叫 `jellyfin-sm`），先停下旧容器再启动，避免冲突：
> ```bash
> docker rm -f jellyfin-sm
> docker compose up -d
> ```

### 验证启动是否成功

```bash
docker logs jellyfin-sm 2>&1 | tail -8
```

看到类似这样一行就对了：

```
2026-07-21 16:51:00 setup: ffmpeg wrapped by spacemaster
```

再打开浏览器访问 `http://<宿主IP>:8096` —— 应进入 **Jellyfin 首次设置向导**（建管理员账号、添加媒体库等）。
按向导走完即可，这部分是 Jellyfin 标准流程，空间大师不干预。

---

## 八、部署网页控制台（端口 8777）

控制台跑在**宿主机**（不是容器里），提供「输入房间尺寸 + 拖动滑块」的网页。

### 1) 放好控制台脚本

```bash
sudo cp peq_toggle_nas.py engine_runtime.py /opt/spacemaster/
sudo chmod +x /opt/spacemaster/peq_toggle_nas.py
```

若主机还没装 Python 3（控制台需要）：

```bash
sudo apt update && sudo apt install -y python3   # Debian/Ubuntu/飞牛
```

### 2) 用 systemd 把它注册成开机自启服务

仓库里已带 `peq-console.service`，复制到系统服务目录并启用：

```bash
sudo cp peq-console.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now peq-console.service
sudo systemctl status peq-console.service
```

最后一行应显示 `active (running)`。

> **没有 systemd 的环境**（如某些 NAS 的容器里、或 macOS）：
> 直接前台/后台跑也行：
> ```bash
> sudo nohup python3 /opt/spacemaster/peq_toggle_nas.py >/opt/spacemaster/peq-console.log 2>&1 &
> ```

## 九、使用：输入房间尺寸并应用

1. 浏览器打开 **`http://<宿主IP>:8777`**，进入校正控制台。
2. 填写你的**房间长、宽、高**（单位：米）。例如 `长 5.2 / 宽 3.8 / 高 2.8`。
3. 选择系统类型：
   - **电视（tv）** —— 折成 2.0 并整体 +6dB 响度补偿；
   - **家庭影院 5.1（ht51）** —— 保持 5.1 多声道直出，只做纯 EQ；
   - **2.1（ht21）** —— 折成 2.1（L+R+LFE 独立）。
4. 拖动滑块试听：
   - 低频 / 中频 / 高频增益（整体音色）；
   - 环绕延时（给环绕声道做距离补偿，让 5.1 更包围）；
   - 左右平衡。
5. 点 **「应用到当前播放」**。

### 生效方式

改动是**热加载**的：你点「应用」后，Jellyfin 的**下一次转码**会自动读取新配置并注入校正滤镜，**无需重启容器、无需刷新播放器**。

回到 Jellyfin（`http://<宿主IP>:8096`）播放任意带音轨的视频，声音即被你的房间校正。

## 十、验证校正是否真的生效

看引擎日志：

```bash
sudo tail -n 20 /opt/spacemaster/wrapper.log
```

- 看到 `WRAP engine build-filter ok af_len=NNN` → 校正滤镜已成功注入，生效了。
- 看到 `WRAP engine binary missing -> transparent passthrough (no DSP)` → 引擎二进制没就位（回到**第四步**检查 `/opt/spacemaster/sm_dsp_engine` 是否存在）。
- 看到 `WRAP input channels=6 layout=...` → 正在处理 5.1 音轨。

## 十一、常见问题（排错）

**Q1：播放正常，但听起来没被校正？**
- 查 `wrapper.log`（第十步）。多半是 `/opt/spacemanager` 里缺 `sm_dsp_engine`，或你还没在控制台点「应用」（此时 `peq_af.txt` 为空，系统会原样透传）。
- 确认控制台 `http://<宿主IP>:8777` 能打开、且已点过「应用」。

**Q2：容器起不来 / 一直重启？**
```bash
docker logs jellyfin-sm 2>&1 | tail -20
```
- 报错提到 `setup.sh` / `No such file` → 确认第四步已把 `setup.sh` 抠进 `/opt/spacemaster`，且 `docker-compose.yml` 里的 `entrypoint` 指向它。
- 端口被占用（如已有旧 Jellyfin 占了 8096）→ 先 `docker rm -f jellyfin-sm` 或相关容器。

**Q3：控制台 8777 打不开？**
```bash
sudo systemctl status peq-console.service
sudo tail -n 20 /opt/spacemaster/peq-console.log
```
- 多半是 Python 没装，或 `peq_toggle_nas.py` / `engine_runtime.py` 没复制到 `/opt/spacemaster`。

**Q4：重启主机后全部复位了？**
不会。Jellyfin 配置在 `./config` 卷，校正配置（房间尺寸、滑块）在 `/opt/spacemaster`，都是持久目录。
只要 `docker compose` 的 `restart: unless-stopped` 与控制台 `systemctl enable` 已设（默认就是），重启后会自动恢复。

**Q5：换了一台新机器 / 重装，怎么最快恢复？**
重跑**第四、六、七、八**步即可。`/opt/spacemaster` 里的配置和引擎都会重新就位。

## 附：peq-console.service 全文

（仓库根目录已附此文件，供第八步第 2) 使用）

```ini
[Unit]
Description=SpaceMaster EQ Console (port 8777)
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
WorkingDirectory=/opt/spacemaster
ExecStart=/usr/bin/python3 /opt/spacemaster/peq_toggle_nas.py
Restart=always
RestartSec=3
User=root
StandardOutput=append:/opt/spacemaster/peq-console.log
StandardError=append:/opt/spacemaster/peq-console.log

[Install]
WantedBy=multi-user.target
```

## 安全与分发说明（实话实说）

- 校正引擎是**编译后的机器码**，不是可读源码；任何人都能从公开镜像 `docker run` 抠出来，这和「直接发编译二进制」是同一层保护。
- 算法**源码** `engine_core.py` 永不离开作者机器，不会进任何公开仓库。
- 你**不需要**给任何人发文件、也不需要自己编译——一切都通过公开的 Docker 镜像分发，这是设计内的最短路径。

# 空间大师 · 发布清单与分发铁律

> 本文件是"哪些文件去哪、为什么"的唯一权威记录。凡改动分发方式，先改这里。

## 一句话规则（铁律）

- **安全可公开的文件 → 上传到公开 git（GitHub 公开仓）**
- **带算法的编译产物 `sm_dsp_engine` → 绝不进 git，烤进 Docker 镜像走 GHCR 公开镜像仓**
- **算法源码 `engine_core.py` → 只在维护者本机，永不离开**

## 为什么这样分

1. **公式 = 核心 IP**，只存在于 `engine_core.py`（源码）与编译后的 `sm_dsp_engine`（机器码）。
2. 编译产物虽可被 `docker cp` 抠出，但已是机器码（非源码），且**随公开镜像统一分发**——用户 `docker compose up` 即得，**零手动发送**（即"公开下载"体验）。
3. 集成层（脚本 / 控制台 / wrapper / 垫片）不含任何公式，可公开审计。

---

## 安全护栏（防误提交）

- `.gitignore` 已排除：`engine_core.py` / `sm_dsp_engine` / `*.bin` / Nuitka 残留（`engine_core.build/` 等）/ `.build-venv/` / 本地配置（`*.local` / `.env`）/ 非 Jellyfin 产品 / 根目录旧厚壳 `/sm_wrapper.sh`。
- **发布永远 `git add <显式文件列表>`，绝不用 `git add .` / `git add -A`**：`.gitignore` 只拦"已列出的模式"，新加闭源文件若忘补 ignore 就会被 `git add .` 误带。
- 提交前自检：`git check-ignore engine_core.py sm_dsp_engine` 必须被拦住（有输出、无报错）。
- **二进制走 Docker 的钥匙**：`.dockerignore` 故意**不**排除 `sm_dsp_engine` → 进构建上下文被 `COPY` 烤入；但 `.gitignore` 排除它 → **永不进 git**。两文件配合，保证"进镜像、不进仓"。

## 维护者构建与分发（速查）

```bash
# 1) 编译（Linux x86_64 宿主 gcc，产物 glibc 兼容宿主+容器）
bash build_engine.sh          # 产出 ./sm_dsp_engine（gitignored，不进 git）

# 2) 烤进镜像
docker build -t ghcr.io/gd-benzz/spacemaster-jellyfin:latest .

# 3) 推公开镜像仓（用户 docker compose up 自动拉，二进制已在镜像内）
echo "$GITHUB_TOKEN" | docker login ghcr.io -u GD-benzz --password-stdin
docker push ghcr.io/gd-benzz/spacemaster-jellyfin:latest

# 4) 公开 git 只提交集成层（显式 add，见上清单）
git add .gitignore .dockerignore README.md 开源说明.md ARCHITECTURE.md 发布清单.md \
        Dockerfile docker-compose.yml docker-jellyfin/sm_wrapper.sh \
        engine_runtime.py peq_toggle_nas.py build_engine.sh
git commit -m "空间大师 Jellyfin 版：开源集成层 + 闭源算法二进制走 Docker 分发"
git push origin main   # 本机/NAS 当前网络打不到 GitHub，需在可达环境执行
```

## 终端用户（零手动拿二进制）

```bash
git clone https://github.com/GD-benzz/spacemaster-jellyfin.git
cd spacemaster-jellyfin
docker compose up -d          # 自动拉公开镜像，二进制已在 /usr/local/bin/sm_dsp_engine
sudo docker cp jellyfin-sm:/usr/local/bin/sm_dsp_engine /opt/spacemaster/sm_dsp_engine
sudo chmod +x /opt/spacemaster/sm_dsp_engine
sudo cp peq_toggle_nas.py engine_runtime.py /opt/spacemaster/
sudo systemctl restart peq-console.service
# 浏览器开 http://<IP>:8777
```

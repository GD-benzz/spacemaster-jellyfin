# 空间大师 · 发布清单与分发铁律

> 本文件是"哪些文件去哪、为什么"的唯一权威记录。凡改动分发方式，先改这里。

## 一句话规则（铁律）

- **安全可公开的文件 → 上传到公开 git（GitHub 公开仓）**
- **带算法的编译产物 `sm_dsp_engine` → 绝不进 git，烤进一个 tiny Docker 镜像（不含 Jellyfin）走 GHCR 公开镜像仓 `ghcr.io/gd-benzz/spacemaster-engine`**
- **算法源码 `engine_core.py` → 只在维护者本机，永不离开**
- **Jellyfin 本身不捆绑**：终端用户自己 `docker pull jellyfin/jellyfin`，我们只发"引擎 + 薄壳"，绝不把整个 Jellyfin 打进我们的镜像

### 🐳 烤进 Docker 镜像（GHCR 公开镜像仓，含编译算法，零 Jellyfin）

| 文件 | 说明 |
|------|------|
| `sm_dsp_engine` | Nuitka 编译的**原生机器码**，**含全部声学算法**（驻波模态 / PEQ / Q增益模型 / 下混系数矩阵 / 对称延时 / 左右平衡）。由维护者在 **Linux x86_64 宿主 gcc** 编译（产物链接宿主 glibc，兼容宿主+容器），放在仓库根目录。被 `.gitignore` 排除、**被 `.dockerignore` 保留**，`docker build` 时由根 `Dockerfile` 烤入镜像的 `/spacemaster/sm_dsp_engine`。 |

- **用户取回方式（零手动发文件）**：
  ```bash
  sudo docker run --rm -v /opt/spacemaster:/out \

    sh -c 'cp /spacemaster/* /out/ && chmod +x /out/sm_dsp_engine /out/setup.sh'
  ```

  把引擎二进制、薄壳、setup 一次性拷到宿主 `/opt/spacemaster`，随后官方 Jellyfin 容器挂载该卷即可用。

## 为什么这样分

1. **公式 = 核心 IP**，只存在于 `engine_core.py`（源码）与编译后的 `sm_dsp_engine`（机器码）。
2. 编译产物虽可被 `docker cp` 抠出，但已是机器码（非源码），且**随公开的 tiny 引擎镜像统一分发**——用户一条 `docker run` 即得，**零手动发送**。
3. **Jellyfin 不捆绑**：我们只是给官方 Jellyfin 加了个校正模块（薄壳 + 引擎二进制放进挂载卷），不该把整套 Jellyfin（系统+.NET+ffmpeg，1.5GB+）烤进我们的镜像。用户自己拉官方 Jellyfin，既省我们镜像体积，也避免版本耦合。
4. 集成层（脚本 / 控制台 / wrapper / 垫片 / setup）不含任何公式，可公开审计。

## 安全护栏（防误提交）

- `.gitignore` 已排除：`engine_core.py` / `sm_dsp_engine` / `*.bin` / Nuitka 残留（`engine_core.build/` 等）/ `.build-venv/` / 本地配置（`*.local` / `.env`）/ 非 Jellyfin 产品 / 根目录旧厚壳 `/sm_wrapper.sh`。
- **发布永远 `git add <显式文件列表>`，绝不用 `git add .` / `git add -A`**：`.gitignore` 只拦"已列出的模式"，新加闭源文件若忘补 ignore 就会被 `git add .` 误带。
- 提交前自检：`git check-ignore engine_core.py sm_dsp_engine` 必须被拦住（有输出、无报错）。
- **二进制走 Docker 的钥匙**：`.dockerignore` 故意**不**排除 `sm_dsp_engine` → 进构建上下文被 `COPY` 烤入；但 `.gitignore` 排除它 → **永不进 git**。两文件配合，保证"进镜像、不进仓"。

## 维护者构建与分发（速查）

```bash
# 1) 编译（Linux x86_64 宿主 gcc，产物 glibc 兼容宿主+容器）
bash build_engine.sh          # 产出 ./sm_dsp_engine（gitignored，不进 git）

# 2) 构建 tiny 引擎镜像（根 Dockerfile：alpine + 烤入 /spacemaster/）
docker build -t ghcr.io/gd-benzz/spacemaster-engine:latest .

# 3) 推送镜像到 GHCR
#    ⚠️ 登录用的 GitHub Token（PAT）必须带 `write:packages` 权限！
#       只有 `repo` 权限的 token 会被拒：
#         denied: permission_denied: The token provided does not match expected scopes.
#       在 https://github.com/settings/tokens 生成 classic token，勾选 write:packages
#       （会自动连带 read:packages）。用户名区分大小写，用 ghcr.io 上的小写名。
echo "$GITHUB_TOKEN" | docker login ghcr.io -u gd-benzz --password-stdin
docker push ghcr.io/gd-benzz/spacemaster-engine:latest
#    🔔 推完务必到 GitHub → Packages → 该包 → Settings/Change visibility 设为 **Public**，
#       否则终端用户 `docker pull` 会 403 拉不到。

# 4) 公开 git 只提交集成层（显式 add，见上清单）
git add .gitignore .dockerignore README.md 开源说明.md ARCHITECTURE.md 发布清单.md 安装指南.md \
        Dockerfile docker-compose.yml docker-jellyfin/sm_wrapper.sh docker-jellyfin/setup.sh \
        engine_runtime.py peq_toggle_nas.py peq-console.service build_engine.sh
git commit -m "空间大师 Jellyfin 版：开源集成层 + 闭源算法二进制走 tiny 引擎镜像分发"
git push origin main
```

> 镜像名必须全小写：`ghcr.io/gd-benzz/spacemaster-engine`。

## 终端用户（零手动拿二进制）

完整逐步教程见 [`安装指南.md`](安装指南.md)。要点：

```bash
git clone https://github.com/GD-benzz/spacemaster-jellyfin.git
cd spacemaster-jellyfin

# 1) 自己拉官方 Jellyfin（我们不捆绑）
docker pull jellyfin/jellyfin:latest

# 2) 从公开 tiny 引擎镜像把引擎/薄壳/setup 取回宿主 /opt/spacemaster
sudo mkdir -p /opt/spacemaster
sudo docker run --rm -v /opt/spacemaster:/out \
  ghcr.io/gd-benzz/spacemaster-engine:latest \
  sh -c 'cp /spacemaster/* /out/ && chmod +x /out/sm_dsp_engine /out/setup.sh'

# 3) 启动（官方 jellyfin + setup.sh 入口，挂载 /opt/spacemaster）
docker compose up -d

# 4) 部署控制台（宿主 systemd，端口 8777）
sudo cp peq_toggle_nas.py engine_runtime.py /opt/spacemaster/
sudo systemctl daemon-reload && sudo systemctl enable --now peq-console.service

# 5) 浏览器开 http://<宿主IP>:8777 输入房间尺寸 → 应用
```

> 缺引擎二进制也不会崩：薄壳会**原样透传音频**（不校正），控制台提示引擎不可用。
